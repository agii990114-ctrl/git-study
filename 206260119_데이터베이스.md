## 데이터베이스
- 이름(컬럼명/ 테이블명) : 물리
- 코멘트 : 논리
  
- 도메인 : 데이터베이스에서 도메인이란 하나의 속성(Attribute)이 가질 수 있는 원자값(Atomic Value)들의 집합을 의미합니다.
- 스키마와 도메인의 관계 (보완 내용)
  - 스키마를 정의할 때 도메인을 정확히 설정해야 하는 이유는 다음과 같습니다.

  - 연산의 적절성: 동일한 도메인에 있는 값들끼리만 비교 연산이 가능합니다. (예: '나이' 도메인과 '이름' 도메인은 비교할 수 없음)

  - 데이터 독립성 강화: 도메인을 별도로 정의해 두면, 여러 속성에서 해당 도메인을 재사용할 수 있어 관리가 효율적입니다.

  - 사용자 관점 (외부 스키마): 사용자가 데이터를 입력할 때, 도메인 설정에 따라 입력 가능한 값이 필터링되어 보여집니다.
