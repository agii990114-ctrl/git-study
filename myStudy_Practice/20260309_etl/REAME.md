# 데이터 처리 개념 정리: ETL & OLTP

## 1. ETL (Extract, Transform, Load)
데이터를 통합하고 분석하기 위한 핵심 프로세스입니다.

* **Extract (추출):** 소스 시스템(DB, API, 파일 등)에서 데이터를 가져오는 단계
* **Transform (변환):** 분석 목적에 맞게 데이터를 정제, 표준화, 결합하는 단계
* **Load (적재):** 가공된 데이터를 데이터 웨어하우스(DW)에 저장하는 단계

> **핵심 가치:** 파편화된 데이터를 하나로 모아 **의사결정의 근거**를 만듦.

---

## 2. OLTP (Online Transaction Processing)
실시간으로 발생하는 비즈니스 트랜잭션을 처리하는 방식입니다.

* **특징:** 빠른 응답 속도, 데이터 무결성 보장, 잦은 수정 및 삭제.
* **주요 작업:** `INSERT`, `UPDATE`, `DELETE`.
* **예시:** 은행 송금, 쇼핑몰 주문, 수강 신청 시스템.

---

## 3. OLTP vs OLAP 비교
데이터베이스 운영 방식에 따른 차이점입니다.

| 구분 | OLTP (트랜잭션 중심) | OLAP (분석 중심) |
| :--- | :--- | :--- |
| **목적** | 일상적인 업무 운영 | 데이터 기반 의사결정 및 분석 |
| **데이터 범위** | 최신 상세 데이터 | 대량의 과거 이력 데이터 |
| **주요 사용자** | 실서비스 사용자, 창구 직원 | 데이터 분석가, 경영진 |
| **성능 지표** | 트랜잭션 처리 속도 (응답성) | 복잡한 쿼리 처리 속도 (처리량) |

---

## 4. 데이터 흐름 아키텍처 (High-Level)
실제 시스템에서는 보통 다음과 같은 순서로 데이터가 흐릅니다.

1.  **OLTP 시스템:** 실시간 서비스 데이터 발생 및 저장 (예: MariaDB)
2.  **ETL 파이프라인:** 운영 DB의 데이터를 추출하여 변환 및 정제 (예: Python, Airflow)
3.  **OLAP / DW:** 분석용 저장소에 데이터 적재 후 시각화 (예: BigQuery, Tableau)
